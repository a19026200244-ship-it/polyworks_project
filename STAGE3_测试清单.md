# Stage 3 测试清单：标定与坐标变换

这份清单对应当前项目的 **Stage 3：点对标定 + 坐标变换 + 机器人结果自动回传机器人坐标系**。

---

## 一、测试目标

验证以下能力是否都正常：

1. 能在 UI 中加载两组点
2. 能求解刚体变换 `R, T`
3. 能显示 `RMS` 和 `最大残差`
4. 能保存一个命名变换
5. 能重新加载并设为当前激活变换
6. 机器人联动在 `FRAME=ROBOT_BASE` 时能自动返回变换后的结果

---

## 二、准备工作

### 1. 安装依赖

```powershell
pip install -r requirements.txt
```

### 2. 启动程序

```powershell
python main.py
```

### 3. 需要用到的数据文件

#### 纯平移示例

- `数据文件/标定_PW_平移示例.txt`
- `数据文件/标定_ROBOT_平移示例.txt`

预期：

- `R = 单位矩阵`
- `T = [100, 200, 300]`
- `RMS ≈ 0`
- `最大残差 ≈ 0`

#### 旋转 + 平移示例

- `数据文件/标定_PW_旋转平移示例.txt`
- `数据文件/标定_ROBOT_旋转平移示例.txt`

预期：

- 旋转约等于 **绕 Z 轴 +90°**
- 平移约等于 `[100, 200, 300]`
- `RMS ≈ 0`
- `最大残差 ≈ 0`

#### 带噪声示例

- `数据文件/标定_PW_带噪声示例.txt`
- `数据文件/标定_ROBOT_带噪声示例.txt`

预期：

- 结果应接近平移 `[10, 20, 30]`
- `RMS > 0`
- `最大残差 > 0`

#### 异常测试示例

- 点数不一致：
  - `数据文件/标定_PW_点数不一致示例.txt`
  - `数据文件/标定_ROBOT_点数不一致示例.txt`
- 退化点集：
  - `数据文件/标定_PW_退化示例.txt`
  - `数据文件/标定_ROBOT_退化示例.txt`

## 三、功能测试清单

---

## 用例 A：纯平移标定

### 步骤

1. 打开程序
2. 进入 **“标定与变换”** 页面
3. 点击 **“加载 PW 点文件”**
   - 选择 `数据文件/标定_PW_平移示例.txt`
4. 点击 **“加载机器人点文件”**
   - 选择 `数据文件/标定_ROBOT_平移示例.txt`
5. 在“变换名称”中输入：
   - `PW_to_ROBOT_BASE_translation_demo`
6. 点击 **“求解刚体变换”**

### 预期结果

- 点对数量显示为 `6`
- `RMS 误差` 接近 `0`
- `最大残差` 接近 `0`
- 平移向量接近：

```text
[100, 200, 300]
```

- 旋转矩阵接近单位矩阵：

```text
[1, 0, 0]
[0, 1, 0]
[0, 0, 1]
```

### 保存验证

7. 点击 **“保存并设为当前”**

### 预期结果

- 弹出保存成功提示
- “当前激活变换”显示为：
  - `PW_to_ROBOT_BASE_translation_demo`

## 用例 D：保存 / 加载变换

### 步骤

1. 先至少保存一个变换
2. 点击 **“刷新列表”**
3. 在“已保存变换”下拉框中选择一个已保存变换
4. 点击 **“加载并设为当前”**

### 预期结果

- 当前激活变换名称更新
- 下方矩阵与平移信息同步刷新

---

## 用例 E：机器人返回 PolyWorks 坐标系结果

### 前提

- 主程序已连接 PolyWorks
- 机器人联动服务已启动
- 当前是否有激活变换都无所谓

### 执行命令

```powershell
python robot_simulator.py --task CIRCLE --frame PW
```

### 预期结果

- 最终返回消息中：

```text
FRAME=PW
```

- 返回的圆心应为 PolyWorks 计算结果本身，不做坐标变换

---

## 用例 F：机器人返回机器人坐标系结果

### 前提

- 已完成一次标定并保存
- 当前已有激活变换
- 主程序已连接 PolyWorks
- 机器人联动服务已启动

### 执行命令

```powershell
python robot_simulator.py --task CIRCLE --frame ROBOT_BASE
```

也可以测试：

```powershell
python robot_simulator.py --task LINE --frame ROBOT_BASE
python robot_simulator.py --task INTERSECTION3P --frame ROBOT_BASE
```

### 预期结果

- 最终返回消息中：

```text
FRAME=ROBOT_BASE
```

- 返回坐标已经经过当前激活变换

---

## 用例 G：无激活变换时请求 ROBOT_BASE

### 前提

- 当前没有激活变换
- 主程序已连接 PolyWorks
- 机器人联动服务已启动

### 执行命令

```powershell
python robot_simulator.py --task CIRCLE --frame ROBOT_BASE
```

### 预期结果

- 服务端应返回错误消息
- 错误码应接近：

```text
CODE=TRANSFORM_ERROR
```

说明：

- 程序知道你要机器人坐标
- 但当前没有可用标定变换

## 用例 H：错误数据测试

### H1：点数不足

做法：

- 把某个标定文件删掉几行，只保留 3 对点

预期：

- UI 求解时报错
- 提示至少需要 4 对点

### H2：两组点数不一致

做法：

- 直接加载：
  - `数据文件/标定_PW_点数不一致示例.txt`
  - `数据文件/标定_ROBOT_点数不一致示例.txt`

预期：

- UI 提示两组点数量不一致

### H3：点退化

做法：

- 直接加载：
  - `数据文件/标定_PW_退化示例.txt`
  - `数据文件/标定_ROBOT_退化示例.txt`

预期：

- 标定求解失败
- 报错提示点集退化或不稳定

## 四、最终验收标准

全部通过后，可以认为 Stage 3 达标：

- [ ] 能加载两组对应点
- [ ] 能求解刚体变换 `R, T`
- [ ] UI 能显示 RMS 和最大残差
- [ ] 能保存一个命名变换
- [ ] 能重新加载并激活变换
- [ ] 机器人联动支持 `FRAME=PW`
- [ ] 机器人联动支持 `FRAME=ROBOT_BASE`
- [ ] 无激活变换时会返回明确错误
- [ ] 日志中能看到标定求解与结果转换记录

---

## 五、建议你给导师演示的顺序

1. 用“纯平移示例”做一次标定
2. 保存并激活变换
3. 运行：

```powershell
python robot_simulator.py --task CIRCLE --frame PW
```

4. 再运行：

```powershell
python robot_simulator.py --task CIRCLE --frame ROBOT_BASE
```

5. 对比两次返回：
   - 第一次是 `PW`
   - 第二次是 `ROBOT_BASE`

这样导师会非常直观地看到：

**“同一个测量结果，已经可以按标定关系转成机器人坐标返回。”**
